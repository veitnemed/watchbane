# Onboarding Reports

This directory is the local output area for onboarding rebuild, quality and
comparison reports.

- `raw/` stores full generated logs, JSON payloads and candidate samples.
- `analysis/` stores short analytical notes and generated markdown summaries.
- `baselines/` stores small snapshots used for before/after comparison.

Do not write generated onboarding reports into `docs/`. Only curated summaries
that should remain part of long-term project documentation belong under
`docs/reports/onboarding/`.
