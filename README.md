# Terminal Movies Learn

Небольшой экспериментальный проект для изучения личного вкуса в фильмах и
сериалах.

Обычные рекомендательные сервисы показывают общий рейтинг: насколько произведение
понравилось большой группе людей. Но высокий рейтинг не означает, что конкретному
человеку будет интересно смотреть сериал. У каждого есть собственные предпочтения:
кому-то важна атмосфера, кому-то динамика, кому-то определённый тип сюжета или
настроение после просмотра.

Идея проекта - собрать личные оценки и попробовать вычислить, какие параметры
лучше всего объясняют именно индивидуальный вкус.

## Как устроен эксперимент

Для каждого фильма или сериала сохраняются:

- личная оценка по десятибалльной шкале;
- рейтинг и количество голосов на Kinopoisk;
- рейтинг и количество голосов на IMDb;
- год выхода;
- субъективные признаки, заданные пользователем.

Субъективные признаки не зашиты в программу навсегда. Их можно добавлять и
удалять через меню. Это могут быть любые критерии, которые хочется проверить:
жанровые элементы, особенности сюжета, настроение, темп, тип персонажей или
пригодность сериала для определённой ситуации.

Например, можно создать признаки:

```text
has_detective_story
has_dark_atmosphere
has_fast_pacing
good_for_evening
```

Для каждого признака задаётся понятное описание и шкала. Новые признаки
автоматически добавляются в датасет и начинают участвовать в обучении.

## Как считается прогноз

Каждая запись превращается в набор чисел. В модель входят:

- оценки Kinopoisk и IMDb;
- популярность, рассчитанная из количества голосов и возраста произведения;
- пользовательские субъективные признаки.

Для каждого параметра хранится вес. Прогноз личной оценки считается как
взвешенная сумма:

```text
прогноз = признак_1 * вес_1 + признак_2 * вес_2 + ... + признак_n * вес_n
```

Обучение модели - это поиск весов, при которых прогнозы как можно меньше
отличаются от реальных личных оценок.

Проект не пытается построить универсальную систему рекомендаций. Это скорее
лаборатория: можно добавлять гипотезы о собственном вкусе и смотреть, помогают
ли они модели точнее предсказывать результат.

## Метрики

Главная метрика - `MAE`, средняя абсолютная ошибка прогноза модели.

```text
MAE = среднее значение |прогноз модели - личная оценка|
```

На главном экране также показывается `KP_MAE`: средняя ошибка рейтинга
Kinopoisk относительно личных оценок.

Сравнение `MAE` и `KP_MAE` отвечает на практический вопрос: удалось ли
персональной модели приблизиться к вкусу пользователя лучше, чем обычному
общему рейтингу.

Обе ошибки отображаются в процентах от десятибалльной шкалы. Чем меньше
значение, тем точнее результат.

## Методы подбора весов

В проекте есть несколько способов обучения. Они решают одну задачу, но
исследуют пространство весов по-разному.

### Последовательный перебор

Модель берёт один параметр и перебирает значения его веса от `0` до `1` с
заданным шагом. Лучшее найденное значение сохраняется, после чего алгоритм
переходит к следующему параметру. Полный проход выполняется несколько раз.

Плюсы:

- простой и понятный алгоритм;
- воспроизводимый результат;
- удобно использовать как базовую точку сравнения.

Ограничение: это не полный перебор всех комбинаций сразу. Значение каждого
веса подбирается по очереди, поэтому алгоритм может остановиться не в глобально
лучшем решении.

### Случайный поиск до плато

Алгоритм случайно выбирает два параметра и переносит небольшую долю веса с
одного на другой. Новая комбинация сохраняется только тогда, когда уменьшает
`MAE`.

Если улучшений нет заданное количество попыток подряд, поиск останавливается.
Этот предел называется плато.

Плюсы:

- сумма весов остаётся нормализованной;
- можно исследовать другие комбинации после последовательного перебора;
- повторные запуски иногда находят дополнительное улучшение.

Ограничение: результат зависит от случайности. Два запуска на одном датасете
могут завершиться с немного разными весами.

### Уточнение с несколькими шагами

Алгоритм несколько раз запускает последовательный перебор, постепенно уменьшая
шаг:

```text
0.05 -> 0.02 -> 0.01 -> 0.005 -> 0.001
```

Сначала крупный шаг быстро находит перспективную область, затем маленькие шаги
уточняют результат.

Это наиболее удобный режим для регулярного использования после пополнения
датасета.

## Как исследовать влияние параметров

После обучения можно посмотреть веса модели. Большой вес означает, что
параметр активно участвует в прогнозе, а вес около нуля - что модель почти его
не использует.

Но один только размер веса не доказывает важность признака. Поэтому есть
отдельная проверка влияния:

1. один параметр временно исключается;
2. остальные веса подбираются заново;
3. ошибка модели без параметра сравнивается с полной моделью.

Если ошибка заметно выросла, параметр действительно полезен. Если почти не
изменилась, признак можно уточнить или заменить. Если ошибка уменьшилась,
признак может добавлять шум.

Так проект позволяет проверять гипотезы: не просто предполагать, что какой-то
критерий важен, а смотреть, улучшает ли он качество модели на реальных оценках.

## Проверка на новых данных

Обычный `MAE` считается на тех же данных, на которых подбирались веса. Поэтому
очень маленькая ошибка сама по себе ещё не гарантирует хороший прогноз для
нового сериала.

Для более честной оценки реализована leave-one-out проверка:

1. один объект временно исключается из датасета;
2. модель обучается на остальных объектах;
3. для исключённого объекта рассчитывается прогноз;
4. процедура повторяется для каждой записи.

В результате можно увидеть среднюю ошибку на исключённых объектах и найти
сериалы, которые модель объясняет хуже всего.

## Что можно обсуждать и улучшать

Проект специально оставлен небольшим и прозрачным. Здесь интересно оценить не
только код, но и сам подход:

- достаточно ли взвешенной суммы для описания личного вкуса;
- нужно ли разрешить отрицательные веса;
- как лучше нормализовать рейтинги и популярность;
- насколько устойчивы веса на маленьком датасете;
- какие субъективные параметры стоит проверять;
- когда новый признак действительно полезен, а когда просто помогает
  переобучиться;
- стоит ли разделять фильмы и сериалы или обучать одну общую модель;
- какие альтернативные методы оптимизации стоит попробовать.

## Возможности программы

- добавление записей через терминальное меню;
- массовое редактирование датасета через Excel;
- хранение данных и весов в JSON;
- резервные копии перед изменением данных;
- добавление и удаление любых субъективных признаков;
- несколько методов подбора весов;
- сравнение персональной модели с рейтингом Kinopoisk;
- анализ влияния отдельных параметров;
- leave-one-out проверка;
- ручной прогноз оценки для нового объекта.

## Запуск

Нужен Python 3 и библиотека `openpyxl`.

```powershell
pip install -r requirements.txt
python main.py
```

При первом запуске программа создаст рабочие файлы и каталоги.

Для запуска встроенных проверок:

```powershell
python test.py
```

## Где лежат данные

Рабочие данные хранятся отдельно от репозитория:

| Назначение | Путь |
| --- | --- |
| Датасет | `C:/DATA/movies-learn/dataset.json` |
| Веса модели | `C:/DATA/movies-learn/weights.json` |
| Метаданные | `C:/META/meta-movies-learn/meta_data.json` |
| Резервные копии | `C:/BACKUP/movies-learn/BACKUP/` |
| Excel для редактирования | `C:/TXT_FILES/movies-learn/edit_dataset.xlsx` |
